# 故障排查向导 · 设计指南

> 供开发对接使用。目标:任何人照此文档 + `troubleshooter-data.json`,都能还原并保持界面一致。以后加产品只改数据文件,不动界面。

---

## 0. 设计目标

- **受众**:工业设备的采购、工程师、现场操作员(B2B)。他们要的是"快速找到答案",不是视觉冲击。
- **基调**:可靠、冷静、专业。像一个安静高效的工程师在旁边带着走。
- **移动优先**:现场人员常拿手机在设备旁边查,触控目标要大,单列自适应。

---

## 1. 配色

主色调:**工业蓝 + 白 + 灰**。语义色**只用于状态**,不滥用。

| 变量 | 色值 | 用途 |
|------|------|------|
| `--brand` | `#1f6feb` | 主色:按钮、链接、高亮、进度 |
| `--brand-dark` | `#1a5fd0` | 主色 hover / 按下 |
| `--brand-tint` | `#eaf2ff` | 主色浅底:选中态、标签背景 |
| `--ink` | `#1a2230` | 主文字 |
| `--ink-soft` | `#4a5568` | 次要文字、说明 |
| `--line` | `#e2e6ec` | 边框、分隔线 |
| `--bg` | `#f6f8fa` | 页面底色 |
| `--card` | `#ffffff` | 卡片 / 面板底色 |

**状态语义色**(对应数据里的 `level`,也和 S2000 光环颜色呼应):

| 变量 | 色值 | 含义 |
|------|------|------|
| `--ok` | `#16a34a` 绿 | 正常 / 无需处理 |
| `--warn` | `#d97706` 黄 | 警告 / 建议尽快处理 |
| `--stop` | `#dc2626` 红 | 严重 / 需立即停机处理 |
| (中性) | `--brand` 蓝 | 继续检查 / 中性追问 |

> **换主色**:只改 `--brand` / `--brand-dark` / `--brand-tint` 三个值,全站跟随。

## 2. 字体与排版

- **字体族**:系统字体栈,免加载、跨平台一致:
  `-apple-system, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`
- **字号**:
  - 页面主问题 `h1.q` — 22px / 粗 700
  - 面板标题 `h2` — 18px / 650
  - 正文 — 14.5px
  - 说明、面包屑、标签 — 12–13px
- **行高**:正文 1.55,标题 1.3
- **字重**:标题 650–700,正文 400,强调/按钮 600–650

---

## 3. 布局

- **容器**:居中,最大宽度 **820px**,左右留白 20px。这个宽度让阅读专注、不散。
- **卡片网格**:`auto-fill, minmax(220px, 1fr)`,自动换行,窄屏自然变单列。
- **圆角**:统一 `--radius: 12px`(小元素 8–10px)。
- **阴影**:轻。`0 1px 3px rgba(20,30,50,.06), 0 6px 20px rgba(20,30,50,.05)`。工业感靠克制,不要重投影。
- **间距**:卡片内 16px;区块间 18–24px。

## 4. 页面结构与流程(三层)

```
顶部 Header(logo + 标签)
面包屑(可点每层回退) + 返回上一步
──────────────────────────────
第1步  选品牌   → 卡片网格(含"待录入"灰色占位,展示可扩展)
第2步  选产品   → 卡片网格
第3步  选症状   → 卡片网格 + 底部错误码速查表(可折叠)
        ↓
排查节点  branch(是非追问) / cause(原因+步骤) / ringpicker(光环颜色)
        ↓
每个 cause 底部:"问题还没解决?→ 联系技术支持 / 询价配件"(留资)
```

**搜索框**:出现在"选品牌"和"选症状"两层顶部。原型未接后端,正式版应接关键词直达(如"error 1""光环红色")。

## 5. 核心组件规范

### 5.1 选择卡片 `.choice`
- 白底、细边、轻阴影、左侧图标 + 右侧标题/描述
- hover:边框变主色 + 上移 1px
- 键盘可聚焦(`:focus-visible` 主色描边)
- **待录入占位** `.soon`:降透明度、不可点、"待录入"小徽章。**故意保留**,让人一眼看懂"加产品=填数据"。

### 5.2 追问 `branch`
- 用面板承载,一句问题 + 一段说明 + 两个大按钮(是 ✓ / 否·不确定)
- 按钮等宽、最小宽 130px,窄屏堆叠

### 5.3 解决方案 `cause`
- 面板**左边框用状态色**(ok/warn/stop/中性),顶部一个状态标签
- 标题 + 一句引导 + **编号步骤列表**(圆形序号,主色)
- 底部灰框 `note` 放关键提示/原理
- 可选 `followup`:再追问一次

### 5.4 光环颜色选择器 `ringpicker`(S2000 专属)
- 5 个色块卡片(红/蓝/品红/黄/绿),点色块直达对应状态
- 这是**亮点交互**:利用产品天然设计,比翻手册直观得多。其他有"状态灯/指示灯"的产品可复用这个形态。

### 5.5 错误码速查表 `ref_tables`
- 症状页底部,**可折叠**(`<details>`),默认收起
- 给愿意自己查的老手兜底,也利于 SEO(错误码是高频搜索词)

### 5.6 留资入口 `.stuck`
- 每个解决方案下方
- 两个动作:**联系技术支持**(主按钮)/ **询价配件耗材**(次按钮)
- **关键**:提交时自动带上"当前 品牌+产品+症状"上下文,直接成为高质量销售线索/工单

---

## 6. 交互规则

- **面包屑**:每一层都可点回退;`toBrand/toProduct/toSymptom` 分别重置下游状态。
- **返回上一步**:始终可用(除第一层)。
- **状态机**:`{ brand, product, symptom, node }` 四个状态驱动渲染,逐层判空。
- **动效**:切换时 `fade`(0.25s 淡入上移)。尊重 `prefers-reduced-motion`,可关闭。
- **无障碍**:卡片/按钮键盘可达;色块不只靠颜色,配文字标签(色盲友好)。

---

## 7. 数据驱动原则(重要)

界面**不写死任何产品内容**,全部读 `troubleshooter-data.json`:

- `brands[]` → 第1、2、3 层的卡片
- `nodes{}` → 每个排查节点(branch/cause/ringpicker)
- `ref_tables{}` → 底部速查表

**加新产品**:在 `brands` 里对应品牌下加一个 product,写它的 symptoms,再在 `nodes` 加它的排查节点即可。**界面代码零改动。**

**加新品牌**:在 `brands[]` 顶层加一条即可(把占位换成真的)。
